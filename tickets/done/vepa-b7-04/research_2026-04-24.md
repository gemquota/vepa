# Research: Law Info UI Expansion (SVGs & Detail)

**Date**: 2026-04-24

## 1. Executive Summary
The Law Info module (formerly Parameter Intel) is currently a simple list of switches and text descriptions. The user wants to add complex SVG icons for each law, increase the complexity/detail of the descriptions, and add a graphic-style introductory section.

## 2. Technical Context
- **UI Logic**: `renderInfoModule` in `src/ui.js` iterates over `pureMap` and `biolMap`.
- **Data Source**: `HELP_DB` in `src/constants.js` contains `hint`, `explanation`, `system`, and `advanced` descriptions.
- **Law Grouping**: Reorganized in ticket `vepa-b7-01` into `PURE PHYSICS` and `BIOLOGICAL TRIGGERS`.

## 3. Findings & Analysis
### Law Icons Requirement
Each of the following needs a unique SVG:
- **Pure Physics**: `grav`, `drag`, `jitter`, `wrap`, `coll`, `accr`.
- **Biology**: `life`, `glow`, `affinity`, `reproduction`, `tracking`, `senescence`, `genotype`, `phenotype`.

### Increased Detail
Current implementation only shows `data.layers.explanation`. To increase complexity, we should surface `system` and `advanced` layers or at least combine them into a more comprehensive block.

### Graphic Introduction
The module needs a header section with an introductory "graphic" description (likely a stylized text block or SVG banner).

## 4. Technical Constraints
- **SVG Complexity**: Hand-coding complex SVGs for 14+ items is time-consuming but necessary for the requested "complex" feel.
- **Scrollable Area**: `info-module-container` has `max-height: 40vh`. With more detail, scrolling will be more prominent.

## 5. Architecture Documentation
- **UI Template Strings**: `ui.js` uses template strings to build the DOM elements.
