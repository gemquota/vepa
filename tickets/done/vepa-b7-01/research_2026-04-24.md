# Research: Biology Expansion and Physics/Biology Law Reorganization

**Date**: 2026-04-24

## 1. Executive Summary
The Vepa engine currently uses a flat structure for "Laws" (simulation toggles) and a categorized structure for DNA. The user wants to reorganize Laws into "pure" (physics) and "biol" (biology) groups and expand biological triggers. Additionally, recent changes have introduced flickering and reduced smoothness in particle movement.

## 2. Technical Context
- **Laws Management**: `VepaEngine.laws` in `src/main.js:37` stores toggles.
- **UI Rendering**: `renderInfoModule` in `src/ui.js:210` generates switches for laws: `grav`, `life`, `drag`, `jitter`, `glow`, `wrap`, `coll`, `accr`.
- **Physics Worker**: `src/worker/physics.worker.js` implements the simulation logic, including sub-stepping (lines 14-17).
- **DNA Configuration**: `DNA_META` and `DNA_RANGES` in `src/constants.js` define species traits.

## 3. Findings & Analysis
### Law Reorganization
The current laws can be mapped as follows:
- **Pure (Physics)**: `grav`, `drag`, `jitter`, `wrap`, `coll`, `accr`.
- **Biol (Biology)**: `life`, `glow`.

The user wants to expand `biol` with:
- `affinity`: Multiplier on gravity based on species (exists in DNA but not as a global law).
- `reproduction`: Split from `life` (mitosis).
- `tracking`: New behavior.
- `senescence`: Aging effects.
- `genotype`: DNA influence.
- `phenotype`: Appearance expression.

### Flickering & Smoothness Issues
- **Sub-stepping**: The current implementation of sub-stepping might be causing temporal aliasing if the step size doesn't align well with the frame rate.
- **Numerical Stability**: The softening factor `2.0` in gravity calculations (`physics.worker.js:68`) prevents singularities but might affect orbital stability.
- **Rendering**: In `src/main.js:296`, sprite visibility and scale are updated every frame. Flickering might occur if `this.particles` is updated asynchronously while rendering is in progress (though `isProcessing` flag is used).

## 4. Technical Constraints
- **Stride**: The particle array uses a stride of 24 (`STRIDE = 24`). Any new per-particle state (like tracking targets) must fit within or expand this stride.
- **Performance**: Sub-stepping increases CPU load in the worker. Large `dt` values lead to many sub-steps (`Math.min(200, ...)`).

## 5. Architecture Documentation
- **Event-based UI**: `ui.js` communicates with `main.js` via a `CustomEvent` bus (`emit` function).
- **DNA Mapping**: DNA indices are hardcoded and mapped via `DNA_META`.
